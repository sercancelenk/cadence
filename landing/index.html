Why Cadence is different

Four trade-offs that productivity tools usually skip. We pick the unfashionable side on each of them.

100% local-first — no cloud account

Every keystroke is written to your machine, not to a server. Atomic fsync'ed writes, 50 rolling auto-snapshots, refuse-to-overwrite guard, enriched in-app recovery UI (per-snapshot counts, Reveal in Finder, restore confirmation). If a boot loads meaningfully less than your last save, an amber banner tells you before you panic. When you want two devices to share a workspace, export an encrypted backup file and merge it in — your bytes stay between your phone and your laptop.

Encrypted at rest on your device

The desktop data file is wrapped in AES-256-GCM with a key derived from your account password via scrypt. Notes get an additional opt-in passphrase lockbox. Portable backups use the same on-device encryption — nothing leaves your machine unless you export it.

AI when you want it

Bring your own API key for Anthropic Claude, OpenAI or Google Gemini. Calls go directly from your device to the provider — no proxy, no Cadence-side telemetry. Turn AI off completely and nothing else changes.

Open source, no upsell

The codebase is on GitHub. No accounts in the cloud, no premium tier, no waitlist. If you don't like a behaviour you can fork it. The desktop app auto-updates straight from GitHub Releases.

Local only · bounded disk use

What "local-first" actually means here

Some "local-first" apps mean "we sync your data through our cloud and call it caching". Cadence is the stricter version.

Where your data lives

On macOS: ~/Library/Application Support/Cadence/. On Windows: %APPDATA%\Cadence\. On Linux: ~/.config/cadence/. Trashing the app does not delete your data — the data folder lives outside the application bundle.

What survives a crash

Every save opens a sibling .tmp, writes, calls fsync, then atomically renames. A power loss mid-write leaves either the old file or the new file — never a half-written one. Worst case: ≤ 400 ms of unflushed typing.

Backups, by default

Every save snapshots the previous file into backups/<userId>/. 50 rolling slots labelled launch / post-login / pre-save / pre-pwchange / pre-restore — oldest pruned automatically so disk use stays bounded. Settings → Backup & recovery combines manual export/import with snapshot restore; each snapshot shows teams / lists / tasks / notes counts and restores with a clear "Loaded: …" confirmation.

Archive without deleting

Notes and to-dos support an Active · Archived segment. Shelving hides items from default views (and archived notes from ⌘K search) while keeping them in your encrypted workspace until you delete them. Settings shows a per-area storage breakdown so you can see what archived content still occupies.

Efficient desktop storage

Large workspaces are split into monthly files on disk for performance, while the base file keeps a full copy for safety if you ever open an older Cadence build. You never need to manage these files — the app reads and writes them automatically.

Integrity check on boot

After each successful save, Cadence remembers how much content you had. If the next launch loads a meaningfully smaller workspace, an amber banner spells out the before/after counts and links straight to recovery — catching "my data vanished" before it becomes a support ticket.

Refuse-to-overwrite

If the data file exists but can't be decrypted with the current session key, the writer refuses. You can't accidentally destroy a locked workspace with one character's worth of state after a bad login.

Device transfer via encrypted backup

There is no cloud sync service in Cadence. To move data between devices, export an encrypted portable ZIP from Settings → Data & backup and merge it on the other device. Merge is additive and idempotent — it never overwrites or deletes what you already have. See the Backup section above for the full story.

No telemetry. Period.

Cadence makes zero network requests on its own. The auto-updater hits GitHub Releases. The AI assistant hits whichever provider you configured. Nothing else dials home.
