Why Cadence is different

Four trade-offs that productivity tools usually skip. We pick the unfashionable side on each of them.

100% local-first, syncs without a cloud

Every keystroke is written to your machine, not to a server. Atomic fsync'ed writes, 50 rolling auto-snapshots, refuse-to-overwrite guard, in-app recovery UI. When you want two devices to share a workspace, an opt-in LAN server lets them talk directly — your bytes stay between your phone and your laptop.

Encrypted on disk, encrypted on the wire

The desktop data file is wrapped in AES-256-GCM with a key derived from your account password via scrypt. Notes get an additional opt-in passphrase lockbox. LAN sync rides over TLS 1.2+ with a self-signed cert and a 192-bit bearer token — bytes are encrypted whether they're sitting still or in motion.

AI when you want it

Bring your own API key for Anthropic Claude, OpenAI or Google Gemini. Calls go directly from your device to the provider — no proxy, no Cadence-side telemetry. Turn AI off completely and nothing else changes.

Open source, no upsell

The codebase is on GitHub. No accounts in the cloud, no premium tier, no waitlist. If you don't like a behaviour you can fork it. The desktop app auto-updates straight from GitHub Releases.

Built for the daily ritual

Cadence isn't trying to replace your IDE, your calendar and your inbox. It's the in-between space — the place you actually plan and reflect.

Notes

A Markdown notebook with a real lockbox.

Two-pane macOS-Notes-style view with a resizable sidebar. Reading defaults to Preview; one click flips into Write mode with a full formatting toolbar (Bold, Italic, Headings, Lists, Code, Code-block, Link, Divider — with ⌘B / ⌘I / ⌘K shortcuts).

Lock individual notes with a workspace passphrase. Notes are encrypted with a non-extractable CryptoKey derived via PBKDF2-SHA-256 (200k iterations) → AES-256-GCM. Strict per-view unlock: every time you open a locked note Cadence asks for the passphrase, even within the same session.

Your devices, in step. No server in the middle.

Every other sync product asks you to trust someone else's cloud. Cadence runs an opt-in HTTPS server inside the desktop app and lets your phone or laptop talk to it directly over your own Wi-Fi. Your bytes never leave your network — encrypted on disk, encrypted on the wire, and never mirrored to a SaaS that could leak, throttle or disappear.

HTTPS by default

Self-signed RSA-2048 / SHA-256 cert generated on the host, persisted on disk, auto-renewed when expiry nears or the LAN IP list changes. TLS 1.2 minimum. Works around HTTPS-Only Mode and Safari Private Relay so a phone PWA can talk to the desktop without mixed-content errors.

Bidirectional auto-sync

Paired devices stay in step in the background — push first, pull after — on launch, on window focus, on visibility change, when the network comes back. SHA-256 ETags decide direction so we never overwrite a remote that moved on, and never re-download bytes you already have.

If-Match · If-None-Match · 304 short-circuit

Optimistic concurrency

The host rejects pushes with a 412 Precondition Failed when an ETag is stale; the app surfaces an explicit "Pull host's version first?" prompt. Last-write-wins becomes a deliberate decision, not an accident. Background auto-sync backs off silently so manual resolution stays the single source of truth.

What "local-first" actually means here

Some "local-first" apps mean "we sync your data through our cloud and call it caching". Cadence is the stricter version.

Where your data lives

On macOS: ~/Library/Application Support/Cadence/. On Windows: %APPDATA%\Cadence\. On Linux: ~/.config/cadence/. Trashing the app does not delete your data — the data folder lives outside the application bundle.

What survives a crash

Every save opens a sibling .tmp, writes, calls fsync, then atomically renames. A power loss mid-write leaves either the old file or the new file — never a half-written one. Worst case: ≤ 400 ms of unflushed typing.

Backups, by default

Every save snapshots the previous file into backups/<userId>/. 50 rolling slots labelled launch / post-login / pre-save / pre-pwchange / pre-restore. Settings → Backups & recovery previews any of them with one click.

Refuse-to-overwrite

If the data file exists but can't be decrypted with the current session key, the writer refuses. You can't accidentally destroy a locked workspace with one character's worth of state after a bad login.

Optional LAN sync (off by default)

Settings → Multi-device sync exposes an HTTPS server (TLS 1.2+, self-signed cert generated on the host) protected by a 192-bit bearer token and an ETag conflict guard. QR-code pairing on phones, bidirectional auto-sync once paired. See the Sync section above for the full story.

No telemetry. Period.

Cadence makes zero network requests on its own. The auto-updater hits GitHub Releases. The AI assistant hits whichever provider you configured. Nothing else dials home.
